
```powershell [Windows]

```

:::

## What is Skylence?

Skylence is a **Claude Code harness builder**. You write `.sky` workflow files; it builds and runs a harness for each matching GitHub event — calling the `claude` CLI, streaming results over WebSocket, and persisting everything locally.

<CardGrid>
  <Card title="Claude-native" icon="rocket">
    Uses `claude` CLI sessions, MCP config, and isolation flags directly — no extra layer
  </Card>
  <Card title="Local-first" icon="laptop">
    No account, no cloud backend. State stored in SQLite on your machine
  </Card>
  <Card title="Structured DAGs" icon="setting">
    Multi-step workflows with dependencies, conditions, loops, and cost caps
  </Card>
  <Card title="Live streaming" icon="document">
    Run events stream over WebSocket to the Skylence SPA in real time
  </Card>
  <Card title="Parse-time safety" icon="approve-check-circle">
    `sky lint` catches injection risks and schema errors before any workflow runs
  </Card>
  <Card title="Self-updating" icon="download">
    `sky update` replaces the binary and bounces the service in one command
  </Card>
</CardGrid>

## Why this exists

I saw a YouTuber demo [Archon](https://archon.diy), a Claude Code harness builder. I tried it myself. Wiring authentication into a simple Laravel todos app took 45–50 minutes end to end. Too slow.

So I built `.sky` — an AI-first format where Claude authors the workflow with tight, performant prompts, but it's still readable enough for a human to review before it ships.
