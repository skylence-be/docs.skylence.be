These are ready-to-use `.sky` workflow files. Place them in `.sky/workflows/` in your repository, then add the corresponding label to a GitHub issue or pull request to trigger them.

Run `sky lint` to validate any workflow before committing.

## Fix a GitHub issue

**Trigger:** Add label `ready-for-sky` to an issue.

Classifies the issue, implements a fix in an isolated branch, runs tests, and opens a draft pull request.

```
⊕meta⊕
name = "fix-github-issue"
description = "Classify issue → implement → test → open draft PR"
trigger.github.events = ["issues.labeled"]
trigger.github.label = "ready-for-sky"
⊕⊕

§classify§
model = "haiku"
§§

∆classify∆
Classify GitHub issue #{{issue.number}}: {{issue.title}}

Body: {{issue.body}}

Output ONLY valid JSON — no other text:
{"issue_type": "bug|feature|enhancement|refactor|chore", "reasoning": "<one sentence>"}
∆∆

§implement§
model = "opus"
effort = "max"
isolation = "worktree"
depends_on = ["classify"]
§§

∆implement∆
Fix issue #{{issue.number}}: {{issue.title}}

Type: $classify.output.issue_type

{{issue.body}}

Implement the fix. Run `make test` after changes. Commit referencing #{{issue.number}}. Push: `git push -u origin HEAD`.
∆∆

§test§
bash = "make test"
depends_on = ["implement"]
§§

§create-pr§
model = "sonnet"
depends_on = ["test"]
§§

∆create-pr∆
Create a draft pull request for issue #{{issue.number}}: {{issue.title}}.

Test output: $test.output

1. Confirm all changes are committed: `git status`
2. Push if not already: `git push -u origin HEAD`
3. Create: `gh pr create --draft --title "<imperative title under 70 chars>" --body "Closes #{{issue.number}}"`
∆∆
```

## Review a pull request

**Trigger:** Add label `sky-review` to a pull request.

Analyzes the diff, runs relevant review checks (code quality, error handling, test coverage), and produces a grouped findings report.

```
⊕meta⊕
name = "smart-pr-review"
description = "Classify PR → run review agents → synthesize findings"
trigger.github.events = ["pull_request.labeled"]
trigger.github.label = "sky-review"
⊕⊕

§classify§
model = "haiku"
§§

∆classify∆
Analyze PR #{{pull_request.number}}: {{pull_request.title}}

Run: `git fetch origin && git diff origin/{{pull_request.base.ref}}...origin/{{pull_request.head.ref}}`

Output ONLY valid JSON — no other text:
{"run_code_review": "true|false", "run_error_handling": "true|false", "run_test_coverage": "true|false", "reasoning": "<brief>"}
∆∆

§code-review§
model = "sonnet"
depends_on = ["classify"]
chain_from = "classify"
when = "$classify.output.run_code_review == 'true'"
§§

∆code-review∆
Review PR #{{pull_request.number}}: {{pull_request.title}}

Run `git fetch origin && git diff origin/{{pull_request.base.ref}}...origin/{{pull_request.head.ref}}`

Check correctness, edge cases, project conventions, and security.
List findings: severity (critical/major/minor), file:line, what to fix.
Conclude: LGTM, LGTM with minor fixes, or NEEDS CHANGES.
∆∆

§synthesize§
model = "sonnet"
depends_on = ["code-review"]
§§

∆synthesize∆
Summarize review findings for PR #{{pull_request.number}}.

Code review: $code-review.output

Group by severity:
- CRITICAL: must fix before merging
- MAJOR: should fix before merging
- MINOR: non-blocking

If none: "LGTM — no fixes required."
∆∆
```

## Implement a written plan

**Trigger:** Add label `sky-implement` to an issue.

The issue body serves as the implementation plan. Skylence executes it step by step in an isolated branch, runs tests, and opens a draft pull request.

```
⊕meta⊕
name = "plan-to-pr"
description = "Execute plan from issue body → test → open draft PR"
trigger.github.events = ["issues.labeled"]
trigger.github.label = "sky-implement"
⊕⊕

§implement§
model = "opus"
effort = "max"
isolation = "worktree"
§§

∆implement∆
Execute the implementation plan for issue #{{issue.number}}: {{issue.title}}

Plan:
{{issue.body}}

Follow the plan step by step. Run `make test` after each logical group of changes. Fix failures before continuing. Commit referencing #{{issue.number}}. Push: `git push -u origin HEAD`.
∆∆

§test§
bash = "make test"
depends_on = ["implement"]
§§

§create-pr§
model = "sonnet"
depends_on = ["test"]
§§

∆create-pr∆
Create a draft pull request for issue #{{issue.number}}: {{issue.title}}.

Test output: $test.output

1. Confirm committed: `git status`
2. Push if needed: `git push -u origin HEAD`
3. Create: `gh pr create --draft --title "<imperative title>" --body "Closes #{{issue.number}}"`
∆∆
```

## Safe refactor

**Trigger:** Add label `sky-refactor` to an issue.

Reads the codebase, plans an ordered refactor with no behavior changes, executes each step in an isolated branch, verifies nothing changed functionally, and opens a draft pull request.

```
⊕meta⊕
name = "refactor-safely"
description = "Analyze → plan → execute in worktree → verify behavior preserved → PR"
trigger.github.events = ["issues.labeled"]
trigger.github.label = "sky-refactor"
⊕⊕

§analyze§
model = "opus"
effort = "max"
§§

∆analyze∆
Analyze the codebase for the refactoring in issue #{{issue.number}}: {{issue.title}}

{{issue.body}}

Read-only analysis — do NOT make any changes yet.

1. Identify target files and functions.
2. Map all call sites across the codebase.
3. Find existing tests for the target code.
4. Identify risks: shared state, exported symbols, module-level side effects.
5. Propose a safe reorganization strategy with rationale.

Produce a structured analysis report.
∆∆

§execute§
model = "opus"
effort = "max"
isolation = "worktree"
depends_on = ["analyze"]
chain_from = "analyze"
§§

∆execute∆
Execute the refactoring for issue #{{issue.number}}: {{issue.title}}

Analysis: $analyze.output

Rules:
- No logic changes — structural moves only, behavior preserved.
- Run `make test` after each step. Fix failures before continuing.
- Commit after each logical task.
- Push when complete: `git push -u origin HEAD`.
∆∆

§test§
bash = "make test"
depends_on = ["execute"]
§§

§create-pr§
model = "sonnet"
depends_on = ["test"]
§§

∆create-pr∆
Create a draft pull request for the refactoring in issue #{{issue.number}}: {{issue.title}}.

Test output: $test.output

1. Confirm committed: `git status`
2. Push if needed: `git push -u origin HEAD`
3. `gh pr create --draft`
   - Title: `refactor: <description>` under 70 chars
   - Body: what was restructured and why, that behavior is preserved, `Closes #{{issue.number}}`
∆∆
```
