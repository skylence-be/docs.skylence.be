Place workflows in `.sky/workflows/` in your repository. Trigger them by adding the specified label to a GitHub issue or pull request.

Run `sky lint` to validate any workflow before committing.

| Workflow | Label trigger | Works on |
|----------|--------------|----------|
| Fix GitHub Issue | `ready-for-sky` | Issue |
| Review Pull Request | `sky-review` | Pull request |
| Implement from Plan | `sky-implement` | Issue |
| Safe Refactor | `sky-refactor` | Issue |
| Resolve Merge Conflicts | `sky-resolve` | Pull request |
| Doc Gardening | `doc-garden` | Issue |
| Triage Issues | `sky-triage` | Issue |

Prompts in these workflows include Mermaid diagrams so Claude can visually reason about the expected flow at runtime.
