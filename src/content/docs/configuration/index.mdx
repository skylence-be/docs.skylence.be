## Precedence

Later sources override earlier ones:

```
defaults → ~/.sky/.env → ~/.sky/config.yaml → environment variables
```

Copy `.env.example` to `.env` for local secrets (e.g. `SKY_GITHUB_SECRET`). Never commit `.env`.

## Config file

`~/.sky/config.yaml` uses YAML key-value pairs matching the setting names below:

```yaml
listen_addr: "0.0.0.0:3090"
log_level: "info"
log_format: "json"
monthly_budget_usd: 50
update_check_on_start: true
```

## TLS

TLS is opt-in per listener. Set both cert and key (PEM paths) to enable HTTPS; setting only one errors at startup.

```yaml
# Main API + WebSocket listener
tls_cert_file: "/etc/sky/tls/cert.pem"
tls_key_file: "/etc/sky/tls/key.pem"

# Bridge listener (port 4747)
bridge_tls_cert_file: "/etc/sky/tls/cert.pem"
bridge_tls_key_file: "/etc/sky/tls/key.pem"
```

## Rate limiting

Per-IP rate limiter applied independently to API, webhook, and WebSocket endpoints. It uses a token bucket — short bursts are allowed, but sustained high traffic is smoothed out. Set RPS to `0` or less to disable a limiter. Burst must be `>= 1` when RPS is positive.

## Metrics

When `metrics_enabled` is `true` (default), a Prometheus exposition endpoint is available. Set `metrics_bind_addr` to expose it on a dedicated address (e.g. `127.0.0.1:9091`); leave empty to serve it on the main listener.
