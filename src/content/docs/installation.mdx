Skylence ships as a single static binary called `sky`. It runs as a local daemon on your own machine - no account, no cloud.

## Install

### macOS / Linux

One command. Auto-detects your OS and architecture, verifies the checksum, installs to `/usr/local/bin/sky`, and runs `sky setup` automatically.

```sh
curl -fsSL https://github.com/skylence-be/skylence/releases/latest/download/install.sh | sh
```

Verify:

```sh
sky --version
```

Skip the automatic setup with `--no-setup` if you want to run it yourself later.

### Installer flags

```sh
# Pin to a specific release
curl -fsSL https://github.com/skylence-be/skylence/releases/latest/download/install.sh | sh -s -- --version v0.1.1

# Install to a user-writable directory (no sudo)
curl -fsSL https://github.com/skylence-be/skylence/releases/latest/download/install.sh | sh -s -- --dir ~/.local/bin
```

The installer supports macOS and Linux on both amd64 and arm64.

### Windows

```powershell
irm https://github.com/skylence-be/skylence/releases/latest/download/install.ps1 | iex
```

Runs `sky setup` automatically at the end. Verify:

```powershell
sky --version
```

Skip auto-setup with `-NoSetup`.

#### Installer flags

```powershell
# Pin to a specific release
& ([ScriptBlock]::Create((irm https://github.com/skylence-be/skylence/releases/latest/download/install.ps1))) -Version v0.1.1

# Install to a custom directory
& ([ScriptBlock]::Create((irm https://github.com/skylence-be/skylence/releases/latest/download/install.ps1))) -Dir C:\tools\sky
```

Requires Windows 11 (PowerShell 5.1+). Installs to `%USERPROFILE%\.sky\bin\sky.exe` and adds it to user PATH.

## Self-update

Once installed, `sky` updates itself from the same release channel:

```sh
sky update --check      # report whether a newer release exists
sky update              # install the latest release (with confirmation prompt)
sky update --to v0.1.1  # pin to a specific tag (rollback or reproducible install)
sky update -y           # install without prompting (CI, scripts)
```

## Run as a service (auto-restart)

To run `sky` as a background service that starts at login and restarts automatically, use the service installer (ships with each release). No admin or `sudo` required - all three platforms install at the user level.

### macOS / Linux

```sh
curl -fsSL https://github.com/skylence-be/skylence/releases/latest/download/install-service.sh | sh
```

- **Linux**: installs a `systemd --user` unit at `~/.config/systemd/user/sky.service` and enables linger so it survives logout.
- **macOS**: writes a LaunchAgent at `~/Library/LaunchAgents/be.skylence.sky.plist` with `KeepAlive` on non-zero exit.

Working directory is `~/.sky`, so `sky` picks up `config.yaml` naturally.

Remove with:

```sh
curl -fsSL https://github.com/skylence-be/skylence/releases/latest/download/install-service.sh | sh -s -- --uninstall
```

### Windows

```powershell
irm https://github.com/skylence-be/skylence/releases/latest/download/install-service.ps1 | iex
```

Registers a user-level Scheduled Task `SkylenceServer` with an `AtLogOn` trigger and a 1-minute restart interval.

Remove with:

```powershell
& ([ScriptBlock]::Create((irm https://github.com/skylence-be/skylence/releases/latest/download/install-service.ps1))) -Uninstall
```

### How it interacts with `sky update`

Once the service is installed, `sky update` records the wrapper type (systemd unit / launchd label / Scheduled Task) in `~/.sky/service.json` and automatically bounces it after a successful binary replace - no "old version still in memory until restart" gap. Opt out per-invocation with `sky update --no-restart-service`.

## Uninstall

Remove the `sky` binary:

```sh
curl -fsSL https://github.com/skylence-be/skylence/releases/latest/download/uninstall.sh | sh
```

By default, your data directory at `~/.sky` (configs, workflow runs, SQLite DB) is preserved. To remove it as well:

```sh
curl -fsSL https://github.com/skylence-be/skylence/releases/latest/download/uninstall.sh | sh -s -- --purge
```

Skip confirmation prompts with `-y`:

```sh
curl -fsSL https://github.com/skylence-be/skylence/releases/latest/download/uninstall.sh | sh -s -- --purge -y
```

If you installed to a custom directory, pass `--dir`:

```sh
curl -fsSL https://github.com/skylence-be/skylence/releases/latest/download/uninstall.sh | sh -s -- --dir ~/.local/bin
```

## Manual install

If you prefer not to pipe a script into `sh`, download the archive for your platform directly.

<Tabs>
    <TabItem label="macOS (arm64)">

Verify the checksum (recommended):

```sh
curl -L -O https://github.com/skylence-be/skylence/releases/latest/download/checksums.txt
shasum -a 256 -c checksums.txt --ignore-missing
```

The line for your archive should end in `OK`. If it doesn't, stop and re-download - do not run the binary.

Browse all releases at [github.com/skylence-be/skylence/releases](https://github.com/skylence-be/skylence/releases).
