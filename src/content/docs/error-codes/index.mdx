---
title: Error Codes
description: All SKY-* error codes with causes and remediation steps.
---

Every non-trivial error in Skylence has a structured code. Codes appear in logs, CLI output, and WebSocket events as `[SKY-XX-NNN] message`.

## Domains

| Prefix | Domain |
|--------|--------|
| `SKY-WF-*` | Workflow — parse, validate, DAG, lint |
| `SKY-RUN-*` | Runner — execution, steps, budget |
| `SKY-WH-*` | Webhooks — ingress, signature, routing |
| `SKY-DB-*` | Database — SQLite store |
| `SKY-CL-*` | Claude CLI — subprocess, streaming |
| `SKY-CFG-*` | Configuration — load, parse, validate |
| `SKY-CLN-*` | Clone — git operations |
| `SKY-AUTH-*` | Authentication / authorization |
| `SKY-WS-*` | WebSocket — connections, subscriptions |
| `SKY-API-*` | REST API — request validation |

## SKY-WF — Workflow

Raised by `sky lint` and the workflow loader. Fix these before a run starts.

| Code | Meaning |
|------|---------|
| `SKY-WF-001` | Workflow parse failure |
| `SKY-WF-002` | Validation failed — missing name, empty steps, etc. |
| `SKY-WF-003` | Workflow name not found |
| `SKY-WF-004` | Failed to read workflow directory or file |
| `SKY-WF-005` | DAG contains a cycle |
| `SKY-WF-006` | Workflow has no steps or nodes |
| `SKY-WF-007` | `.sky` section format error — unclosed delimiter, orphan prompt, nested section, malformed marker, or name collision |
| `SKY-WF-010` | Command `.md` file not found in `.sky/commands/` |
| `SKY-WF-011` | Command file parse error |
| `SKY-WF-012` | Command file has no prompt body |
| `SKY-WF-020` | Duplicate node ID in the workflow |
| `SKY-WF-021` | `depends_on` references an unknown node |
| `SKY-WF-022` | Node has no action — needs one of `prompt`, `command`, `bash`, `http`, or `eval` |
| `SKY-WF-023` | Node depends on itself |
| `SKY-WF-024` | Node has more than one execution kind set |
| `SKY-WF-030` | Invalid `$node.output` reference |
| `SKY-WF-031` | Failed to parse `when` condition expression |
| `SKY-WF-032` | Invalid `trigger_rule` value |
| `SKY-WF-033` | Invalid `output_style` value |
| `SKY-WF-034` | `http` node: `url` is required |
| `SKY-WF-035` | `eval` node: `source` is required |
| `SKY-WF-036` | `eval` node: exactly one assertion required (`contains`, `matches`, or `equals`) |
| `SKY-WF-037` | `loop` body cannot be an `http`, `eval`, or `wait` node |
| `SKY-WF-038` | `loop.until` requires exactly one of `bash` or `eval` |
| `SKY-WF-039` | `wait.channel` is not a valid value |
| `SKY-WF-040` | `wait.timeout` is not a valid duration |
| `SKY-WF-041` | `chain_from` is only valid on `prompt` or `command` nodes |
| `SKY-WF-042` | `chain_from` target is not listed in `depends_on` |
| `SKY-WF-043` | `chain_from` target is not a `prompt` or `command` node |
| `SKY-WF-044` | Workflow exceeds the maximum node count (256) |
| `SKY-WF-045` | JSON Schema validation failure |
| `SKY-WF-046` | `loop.max` exceeds the allowed cap |
| `SKY-WF-047` | Bare `{{var}}` inside a JSON string literal in `http.body` — use `{{var\|json}}` instead |
| `SKY-WF-048` | Bash sink (`eval`/`source`/`bash -c`) receives `$SKY_OUTPUT_<prompt-node>` — command injection risk |
| `SKY-WF-049` | `mcp_servers` entry has invalid shape — missing `command`, `url`, or `type` |
| `SKY-WF-050` | `mcp_servers` name collides with a managed server |
| `SKY-WF-051` | `isolation` has an unsupported value |
