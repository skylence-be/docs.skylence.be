---
title: Foutcodes
description: Alle SKY-* foutcodes met oorzaken en oplossingen.
---

Elke niet-triviale fout in Skylence heeft een gestructureerde code. Codes verschijnen in logs, CLI-uitvoer en WebSocket-events als `[SKY-XX-NNN] bericht`.

## Domeinen

| Prefix | Domein |
|--------|--------|
| `SKY-WF-*` | Workflow - parseren, valideren, DAG, lint |
| `SKY-RUN-*` | Runner - uitvoering, stappen, budget |
| `SKY-WH-*` | Webhooks - ontvangst, handtekening, routing |
| `SKY-DB-*` | Database - SQLite-opslag |
| `SKY-CL-*` | Claude CLI - subproces, streaming |
| `SKY-CFG-*` | Configuratie - laden, parseren, valideren |
| `SKY-CLN-*` | Clone - git-operaties |
| `SKY-AUTH-*` | Authenticatie / autorisatie |
| `SKY-WS-*` | WebSocket - verbindingen, abonnementen |
| `SKY-API-*` | REST API - verzoekvalidatie |

---

## SKY-WF - Workflow

Gegenereerd door `sky lint` en de workflow-loader. Los deze op voordat een run start.

| Code | Betekenis |
|------|-----------|
| `SKY-WF-001` | Workflow-parsefout |
| `SKY-WF-002` | Validatie mislukt - ontbrekende naam, lege stappen, etc. |
| `SKY-WF-003` | Workflownaam niet gevonden |
| `SKY-WF-004` | Kan workflowmap of -bestand niet lezen |
| `SKY-WF-005` | DAG bevat een cyclus |
| `SKY-WF-006` | Workflow heeft geen stappen of nodes |
| `SKY-WF-007` | `.sky`-sectieformaatfout - niet gesloten scheidingsteken, zwevende prompt, geneste sectie, misvormd markering of naamconflict |
| `SKY-WF-010` | Commando-`.md`-bestand niet gevonden in `.sky/commands/` |
| `SKY-WF-011` | Commando-bestand parsefout |
| `SKY-WF-012` | Commando-bestand heeft geen prompttekst |
| `SKY-WF-020` | Dubbel node-ID in de workflow |
| `SKY-WF-021` | `depends_on` verwijst naar een onbekende node |
| `SKY-WF-022` | Node heeft geen actie - vereist één van `prompt`, `command`, `bash`, `http` of `eval` |
| `SKY-WF-023` | Node is afhankelijk van zichzelf |
| `SKY-WF-024` | Node heeft meer dan één uitvoeringssoort ingesteld |
| `SKY-WF-030` | Ongeldige `$node.output`-verwijzing |
| `SKY-WF-031` | Kan `when`-conditie-expressie niet parseren |
| `SKY-WF-032` | Ongeldige `trigger_rule`-waarde |
| `SKY-WF-033` | Ongeldige `output_style`-waarde |
| `SKY-WF-034` | `http`-node: `url` is vereist |
| `SKY-WF-035` | `eval`-node: `source` is vereist |
| `SKY-WF-036` | `eval`-node: precies één bewering vereist (`contains`, `matches` of `equals`) |
| `SKY-WF-037` | `loop`-body kan geen `http`-, `eval`- of `wait`-node zijn |
| `SKY-WF-038` | `loop.until` vereist precies één van `bash` of `eval` |
| `SKY-WF-039` | `wait.channel` is geen geldige waarde |
| `SKY-WF-040` | `wait.timeout` is geen geldige duur |
| `SKY-WF-041` | `chain_from` is alleen geldig op `prompt`- of `command`-nodes |
| `SKY-WF-042` | `chain_from`-doel staat niet in `depends_on` |
| `SKY-WF-043` | `chain_from`-doel is geen `prompt`- of `command`-node |
| `SKY-WF-044` | Workflow overschrijdt het maximale aantal nodes (256) |
| `SKY-WF-045` | JSON Schema-validatiefout |
| `SKY-WF-046` | `loop.max` overschrijdt de toegestane limiet |
| `SKY-WF-047` | Kale `{{var}}` in een JSON-stringliteraal in `http.body` - gebruik `{{var\|json}}` |
| `SKY-WF-048` | Bash-sink (`eval`/`source`/`bash -c`) ontvangt `$SKY_OUTPUT_<prompt-node>` - command injection-risico |
| `SKY-WF-049` | `mcp_servers`-entry heeft ongeldige vorm - ontbreekt `command`, `url` of `type` |
| `SKY-WF-050` | `mcp_servers`-naam botst met een beheerde server |
| `SKY-WF-051` | `isolation` heeft een niet-ondersteunde waarde |
