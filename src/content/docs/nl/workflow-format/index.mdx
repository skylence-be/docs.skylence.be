`.sky`-bestanden zijn Go-geparseerde workflowdefinities. LLMs schrijven ze; mensen beoordelen wat de LLM heeft geschreven.

Workflows staan in `.sky/workflows/*.sky` in je repository.

---

## Hoe een .sky-bestand is opgebouwd

Een `.sky`-bestand wordt opgedeeld in secties door vier **Unicode-scheidingstekens** — gekozen omdat ze nooit voorkomen in normale code of tekst, zodat escaping niet nodig is.

Elk scheidingsteken omsluit een benoemd blok. De naam (het **id**) is wat alles aan elkaar koppelt: `§plan§` declareert een stap, en `∆plan∆` geeft de instructies die Claude voor precies die stap leest.

Er zijn vier soorten blok:

<div class="sky-box sky-box-meta">
  <strong><span class="sky-sym">⊕</span><span class="sky-id">meta</span><span class="sky-sym">⊕</span> … <span class="sky-sym">⊕⊕</span></strong> — <Badge text="verplicht · precies één" variant="note" />
  <p>De workflow-header. Definieert de naam, welke gebeurtenis hem triggert en globale instellingen (MCP-servers, budgetlimieten). Go leest dit. Claude niet.</p>
</div>

<div class="sky-box sky-box-step">
  <strong><span class="sky-sym">§</span><span class="sky-id">id</span><span class="sky-sym">§</span> … <span class="sky-sym">§§</span></strong> — <Badge text="één per stap" variant="caution" />
  <p>Stapconfiguratieblok. Declareert welk model te gebruiken, van welke andere stappen deze afhangt, of in een geïsoleerde worktree wordt uitgevoerd, budgetlimieten en meer. Go leest dit om de uitvoeringsgraph te bouwen. Claude ziet deze sleutels niet.</p>
</div>

<div class="sky-box sky-box-prompt">
  <strong><span class="sky-sym">∆</span><span class="sky-id">id</span><span class="sky-sym">∆</span> … <span class="sky-sym">∆∆</span></strong> — <Badge text="optioneel · gekoppeld via id" variant="tip" />
  <p>Prompttekst. Dit is wat Claude daadwerkelijk leest. Gekoppeld aan zijn stap via hetzelfde id. Templatevariabelen zoals <code>{"{{issue.number}}"}</code> worden vervangen voordat Claude de tekst ziet. Geen stapconfig hier — alleen instructies.</p>
</div>

<div class="sky-box" style="border-color: #94a3b8; background: #94a3b810;">
  <strong><span class="sky-sym">※※</span> … <span class="sky-sym">※※</span></strong> — <Badge text="optioneel · genegeerd" variant="default" />
  <p>Docblok. Parser negeert alle inhoud tussen de twee <code>※※</code>-markeringen. Gebruik overal tussen secties om uit te leggen wat de workflow of stap doet — geen effect op uitvoering, geen tokens naar Claude.</p>
</div>

<Aside type="note" title="Uitvoeringsvolgorde komt van depends_on">
Volgorde van boven naar onder is de standaard voor stappen zonder gedeclareerde afhankelijkheden. Voeg `depends_on = ["stap-id"]` toe om een echte volgorde te vereisen. De runner bouwt een DAG en voert stappen parallel uit waar mogelijk.
</Aside>

Elke opener en closer moet **alleen op zijn eigen regel** staan.

## Wat elke sectie kan bevatten

### <span class="sky-sym">⊕meta⊕</span> — workflow-identiteit

| Sleutel | Doel |
|---------|------|
| `name` | Unieke workflownaam (gebruikt in logs en runhistorie) |
| `trigger.github.events` | GitHub-webhookgebeurtenissen die deze workflow activeren |
| `trigger.github.label` | Labelfilter (alleen issues/PR's) |
| `trigger.github.repo` | Beperken tot specifieke repo (`owner/repo`) |
| `max_budget_usd` | Globale budgetlimiet voor alle stappen |
| `mcp_servers` | MCP-servermap — samengevoegd met `~/.sky/claude-env/mcp.json` |

### <span class="sky-sym">§</span><span class="sky-id">id</span><span class="sky-sym">§</span> — stapconfig

`key = value`-paren, één per regel. Waarden zijn JSON-strings, getallen, booleans, `null` of JSON-arrays. Regels die beginnen met `#` zijn commentaar.

| Sleutel | Type | Beschrijving |
|---------|------|--------------|
| `command` | string | Verwijzing naar `.sky/commands/<naam>.md` |
| `prompt` | string | Inline prompt (vult ook `command` aan) |
| `bash` | string | Uit te voeren shellcommando |
| `http` | object | Configuratie voor uitgaande HTTP-call |
| `eval` | object | Bewering op output van eerdere stap |
| `loop` | object | Herhaal body totdat conditie slaagt |
| `wait` | object | Pauzeer voor menselijke goedkeuring of webhook |
| `depends_on` | array | Stap-ID's waarop deze stap wacht |
| `when` | string | Conditie-expressie — sla stap over als onwaar |
| `model` | string | Claude-model: `sonnet`, `opus`, `haiku` |
| `effort` | string | Claude-inspanningsniveau |
| `isolation` | string | `worktree` — voer uit in geïsoleerde git worktree |
| `max_budget_usd` | number | Maximale uitgavengrens per stap |
| `trigger_rule` | string | `all_done` (standaard), `all_success`, `one_success`, `one_failure` |
| `chain_from` | string | Stap-ID waarvan de Claude-sessie wordt hervat |
| `output_format` | object | JSON-schema voor gestructureerde output |

### <span class="sky-sym">∆</span><span class="sky-id">id</span><span class="sky-sym">∆</span> — prompttekst

Vrije tekst. Claude leest dit letterlijk (na vervanging van templatevariabelen). Geen speciale key=value-syntaxis — alleen instructies. Kan meerdere alinea's bevatten, codeblokken, Mermaid-diagrammen of andere inhoud die Claude helpt de taak te begrijpen.

## Templatevariabelen

Gebruik `{{var}}` in `∆`-promptteksten en `http`-nodeconfiguraties om webhook-payloadvelden te vervangen.

```
∆begroet∆
Zeg hallo. Issuenummer is {{issue.number}} in {{repo.full_name}}.
∆∆
```

### Filters

| Filter | Output |
|--------|--------|
| `{{var}}` | Ruwe waarde |
| `{{var\|json}}` | JSON-gecodeerde waarde — gebruik in JSON-bodies zonder aanhalingstekens |
| `{{var\|urlencode}}` | URL-gecodeerde waarde — gebruik in querystrings |

`sky lint` markeert kale `{{var}}` binnen JSON-stringliterals als `SKY-WF-047`.

### Bash-nodes

`{{var}}` is **niet beschikbaar** in `bash` of `loop.until.bash`. Webhook-variabelen bereiken bash via omgevingsvariabelen: `SKY_<HOOFDLETTER_SLEUTEL>` (punten → underscores). Eerdere stapoutputs zijn beschikbaar als `$SKY_OUTPUT_<STAP_ID>`. Altijd quoten: `"$SKY_ISSUE_NUMBER"`.

## MCP-servers

Stel MCP-servers beschikbaar aan alle prompt/command-stappen via `mcp_servers` in `⊕meta⊕`:

```
⊕meta⊕
name = "with-gitnexus"
mcp_servers = {"gitnexus": {"command": "npx", "args": ["gitnexus"]}}
⊕⊕
```

De map wordt samengevoegd met `~/.sky/claude-env/mcp.json`. Naambotsingen met beheerde servers breken de run af.

---

## Validatie

```sh
sky lint
```

Valideert alle `.sky`-bestanden in de huidige repository. Voer dit uit vóór het committen van workflowwijzigingen.
