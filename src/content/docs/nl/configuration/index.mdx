## Prioriteit

Latere bronnen overschrijven eerdere:

```
standaardinstellingen → ~/.sky/.env → ~/.sky/config.yaml → omgevingsvariabelen
```

Kopieer `.env.example` naar `.env` voor lokale secrets (bijv. `SKY_GITHUB_SECRET`). Commit `.env` nooit.

## Configuratiebestand

`~/.sky/config.yaml` gebruikt YAML sleutel-waardeparen die overeenkomen met de namen hieronder:

```yaml
listen_addr: "0.0.0.0:3090"
log_level: "info"
log_format: "json"
monthly_budget_usd: 50
update_check_on_start: true
```

## TLS

TLS is opt-in per listener. Stel zowel cert als key (PEM-paden) in om HTTPS te activeren; slechts één instellen geeft een fout bij opstarten.

```yaml
# Hoofd-API + WebSocket-listener
tls_cert_file: "/etc/sky/tls/cert.pem"
tls_key_file: "/etc/sky/tls/key.pem"

# Bridge-listener (poort 4747)
bridge_tls_cert_file: "/etc/sky/tls/cert.pem"
bridge_tls_key_file: "/etc/sky/tls/key.pem"
```

## Snelheidsbeperking

Per-IP-limiter, afzonderlijk toegepast op API-, webhook- en WebSocket-eindpunten. Er wordt een token bucket gebruikt - korte pieken zijn toegestaan, maar aanhoudend hoog verkeer wordt afgevlakt. Stel RPS in op `0` of minder om een limiter uit te schakelen. Burst moet `>= 1` zijn wanneer RPS positief is.

## Metrics

Wanneer `metrics_enabled` `true` is (standaard), is een Prometheus-expositioneindpunt beschikbaar. Stel `metrics_bind_addr` in om het op een apart adres te bedienen (bijv. `127.0.0.1:9091`); laat leeg om het op de hoofd-listener te serveren.
