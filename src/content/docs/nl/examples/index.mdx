Plaats workflows in `.sky/workflows/` in je repository. Activeer ze door het opgegeven label toe te voegen aan een GitHub-issue of pull request.

Voer `sky lint` uit om een workflow te valideren vóór het committen.

| Workflow | Label-trigger | Werkt op |
|----------|--------------|----------|
| GitHub-issue oplossen | `ready-for-sky` | Issue |
| Pull request beoordelen | `sky-review` | Pull request |
| Plan implementeren | `sky-implement` | Issue |
| Veilige refactoring | `sky-refactor` | Issue |
| Merge-conflicten oplossen | `sky-resolve` | Pull request |
| Doc Gardening | `doc-garden` | Issue |
| Issues triageren | `sky-triage` | Issue |

Prompts in deze workflows bevatten Mermaid-diagrammen zodat Claude de verwachte flow visueel kan begrijpen tijdens uitvoering.
